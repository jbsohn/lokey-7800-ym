# YmSound Sound Effects Format Specification

This document specifies the input and output formats for the `YmSound` sound effects (SFX) compiler in the Lokey 7800 YM project. 

---

## 1. The AYFX CSV Scheme (GUI Visual Export)

This format represents the raw comma-separated value (CSV) export generated by **AYFXedit**. Each line represents a single audio frame (typically updating at 50Hz or 60Hz).

### Column Definition

A valid CSV frame consists of **exactly 5 columns**:

```csv
ToneEnable,NoiseEnable,TonePeriod,NoisePeriod,Volume
```

| Column | Name | Type | Value Range | Description |
| :--- | :--- | :--- | :--- | :--- |
| **1** | `ToneEnable` | Binary Flag | `0` or `1` | Tone generator enable (`1` = ON/Enabled, `0` = OFF/Disabled). *Note: This is active-high in CSV, opposite of the physical PSG register.* |
| **2** | `NoiseEnable` | Binary Flag | `0` or `1` | Noise generator enable (`1` = ON/Enabled, `0` = OFF/Disabled). *Note: Active-high.* |
| **3** | `TonePeriod` | Hex/Int | `0x000` to `0xFFF` | 12-bit tone wavelength divider. Lower values yield higher pitches. |
| **4** | `NoisePeriod` | Hex/Int | `0x00` to `0x1F` | 5-bit noise wavelength divider (0–31). |
| **5** | `Volume` | Hex/Int | `0x0` to `0xF` | 4-bit channel amplitude volume (0–15). |

### Example CSV Data (`explosion.csv`)
```csv
0,1,0x8a8,0x1f,0xf
0,1,0x8a8,0x1c,0xe
0,0,0x8a8,0x19,0x0
0,1,0x8a8,0x15,0xc
```

## 2. The YMS YAML / JSON Scheme (Hand-Authoring)

This format represents the **`.yms`** source format, which is the human-readable, self-documenting format designed for hand-editing in code editors. It supports **metadata headers** and **frame-to-frame delta-omission** (carrying forward unchanged values).

### Schema Structure

```yaml
# yaml-language-server: $schema=./sfx-schema.json
name: string                # Unique identifier for assembly constants
sourceClock: integer        # Designer's host PSG clock (e.g., 2000000)
sourceHz: integer           # Designer's host tick rate (e.g., 50 or 60)
priority: integer           # 0 (lowest) to 255 (highest) for replayer arbitration
preferredChannels: array    # (Optional) List of channels: ["A", "B", "C"]
loopStart: integer/null     # (Optional) Frame index to loop back to; null for one-shot
frames: list of objects     # Frame list
```

### Frame Properties (Delta-Encoded)

All frame properties are **optional**. If a key is omitted, the compiler carries forward the value from the previous frame.

*   **`toneEnable`** (boolean): `true` (ON) or `false` (OFF).
*   **`noiseEnable`** (boolean): `true` (ON) or `false` (OFF).
*   **`tone`** (integer, `0–4095`): 12-bit tone period.
*   **`noise`** (integer, `0–31`): 5-bit noise period.
*   **`volume`** (integer, `0–15`): 4-bit volume.
*   **`duration`** (integer, default `1`): Number of ticks to hold this frame.

### Example YAML Data (`laser.yaml`)
```yaml
# yaml-language-server: $schema=./sfx-schema.json
name: sfx_laser
sourceClock: 2000000
sourceHz: 50
priority: 128
loopStart: null
frames:
  - toneEnable: true
    noiseEnable: false
    tone: 450
    volume: 15
  - tone: 500
    volume: 12      # toneEnable and noiseEnable are carried forward (true/false)
  - tone: 600
    volume: 9
  - volume: 0       # Silence
```

## 3. Target Cartridge Binary Scheme (Compiled Output)

This is the compact byte format (`.ysb`) generated by the Rust compiler for integration into the 7800 cartridge ROM.

### Frame Compression Format

To save bank space, each compiled frame uses a **1-byte bitmask** indicating which register values follow:

```text
┌───────────────────────────────────────────────────────────────┐
│                          FRAME MASK                           │
├───────┬───────┬───────┬───────┬───────┬───────┬───────┬───────┤
│ Bit 7 │ Bit 6 │ Bit 5 │ Bit 4 │ Bit 3 │ Bit 2 │ Bit 1 │ Bit 0 │
│ Dur   │ Dur   │ Noise │ Noise │ Tone  │ Vol   │ Pitch │ Pitch │
│  (H)  │  (L)  │ Period│ Enable│ Enable│       │  (H)  │  (L)  │
└───────┴───────┴───────┴───────┴───────┴───────┴───────┴───────┘
```

#### Mask Bits:
*   **Bit 0**: Pitch Low byte changed (R0/R2/R4) $\rightarrow$ 1 byte follows.
*   **Bit 1**: Pitch High byte changed (R1/R3/R5) $\rightarrow$ 1 byte follows.
*   **Bit 2**: Volume changed (R8/R9/R10) $\rightarrow$ 1 byte follows.
*   **Bit 3**: Tone Enable changed (R7 bit 0/1/2) $\rightarrow$ No byte follows (value derived from mask).
*   **Bit 4**: Noise Enable changed (R7 bit 3/4/5) $\rightarrow$ No byte follows (value derived from mask).
*   **Bit 5**: Noise Period changed (R6) $\rightarrow$ 1 byte follows.
*   **Bits 6–7**: Frame Duration (`00` = 1 tick, `01` = 2 ticks, `10` = 3 ticks, `11` = 4+ ticks).
